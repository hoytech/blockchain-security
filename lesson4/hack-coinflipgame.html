Lab 5: Hack Coin Flip Game

Write a single-use solidity contract to steal funds from the CoinFlipGame contract you made in the previous lab. Your submission should be the solidity program.
  • There are several ways to do this. Note that a single-use contract can check conditions before and/or after it calls another contract. Also, understand that your single-use contract transaction is executing in the exact same block (in fact the same transaction) as the calls it makes to other contracts.
  • Use the following interface: 
    interface CoinFlipGameInterface {
    function bet() external payable;
    function maxBet() external view returns(uint256);
}
  • Since the maximum bet size is 1/10th the contract balance, you can't fully empty it. Try to make your hack take as much as it can using the fewest transactions possible.
  • Bonus: How many wins of maximum bet size does it take to get at least 90% of the balance? How about X%?
